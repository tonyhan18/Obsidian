# 20260407 기술 학습 요약

## 1. 프론트엔드 스타일링 심화

• CSS 박스 모델: `background-color`는 패딩 영역까지 채우지만 마진(Margin) 영역은 채울 수 없음을 확인. 마진 영역에 색을 입히려면 부모의 패딩으로 전환하거나 `box-shadow`를 활용하는 우회 전략 학습.

• SVG 아이콘 관리:

  • `` 방식: 캐싱에 유리하나 CSS 색상 제어 불가.
  • 인라인 `` 방식: CSS 제어는 완벽하나 코드 가독성 저해.
  • 결론: `SVGR` 설정을 통해 파일을 분리하면서도 React 컴포넌트처럼 불러와 사용해 가독성과 제어력을 모두 잡는 방식이 실무 정석임을 파악.

## 2. 웹 폰트 아키텍처 (Container-Presenter)

• @font-face 역할: 브라우저에 폰트 정보를 등록하는 '출생신고서' 역할 및 굵기 별 매핑 원리 이해.

• [Next.js](http://next.js/) 최적화: `next/font`를 사용하여 레이아웃 상단에서 폰트를 로드하고 CSS 변수(`--font-x`)로 주입하여 CLS(Layout Shift) 방지.

• 관심사 분리: Container는 폰트를 몰라야 하며, Presenter는 글로벌 SCSS Mixin을 통해 정의된 타이포그래피 시스템을 소비하도록 설계.

## 3. 개발자 성장 및 커리어

• 추천 도서: 『우아한 타입스크립트 with 리액트』, 『리액트 핵심 UI 패턴』 등 실무 아키텍처 중심의 학습 로드맵 설정.
• 마인드셋: 디자인 감각과 엔지니어링 역량을 분리하여, 시스템을 조립하고 자동화하는 '프론트엔드 엔지니어'로서의 정체성 강화.
